lambda的使用

简介

lambda表达式 () -> {方法体}。（只有一个参数时括号可以省略）

语法格式：java8中出了一个全新的操作符“->“，该操作符称为箭头操作符或lambda操作符

lambda操作符将lambda表达式分为两个部分：

左侧：函数式接口中抽象方法的参数列表

右侧：函数式接口中抽象方法的实现，即lambda体

lambda表达式需要函数式接口的支持

函数式接口

函数式接口：接口中只有一个抽象方法的接口，称为函数式接口。函数式接口可以使用@FunctionalInterface进行注解。说明该接口必须是函数式接口

例如：

public interface Myinterface{

​ public void show();

}

接口中只有一个抽象方法，该接口可以被认定为函数式接口

具体语法

lambda表达式就是对函数式接口抽象方法的实现

  • 语法一：无参数无返回

​ () -> system.out.println(“lambda表达式”);

  • 语法二：有一个参数无返回值

​ (s) -> system.out.println(s); s -> system.out.println(s);

  • 语法三：有一个参数无返回值（只有一个参数时括号可以省略）

​ s -> system.out.println(s);

  • 语法四：有多个参数，有返回值，多个语句

​ (x,y) ->{

​ system.out.prinln(“lambda有多条语句”);

​ retrun Integer.compare(x.length(),y.length());

​ };

  • 语法五：有多个参数，有返回值，只有一条语句 大括号和return都可以省略

​ (x,y) -> Integer.compare(x.length(),y.length());

  • 语法六：lambda表达式参数列表中的数据类型可以省略不写，因为java编译器可以通过上下文推断出对应的数据类型，即“类型推断”

方法引用

简介

方法引用：若Lambda表达式的Lambda体中，需要实现的功能，已经有方法提供了实现，我们可以选择使用方法引用（可以理解方法引用是Lambda表达式 的另外一种表现形式）

格式

方法引用的格式：

  • 对象的引用::实例方法名

  • 类名::静态方法名

  • 类名::实例方法名

注意事项

  • 方法引用中方法的参数列表与返回值类型必须与函数式接口中 抽象方法的参数列表和返回值类型保持一致
  • 返回值类型相同的情况下，函数式接口中参数列表的第一个参数，是方法引用方法的调用者，第二个参数（或无参数）是方法引用方法的参数时，可以使用 类名::实例方法名
